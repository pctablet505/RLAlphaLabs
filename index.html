The Problem

Algo Trading is Hard.
Most Fail in Production.

Building a strategy that looks great in backtests is easy. Making it survive live markets is not.

01

Curve Fitting

A model optimized on historical data captures noise, not signal. Without walk-forward validation, backtest performance is fiction.

02

Unrealistic Simulation

Ignoring NSE transaction taxes (STT, stamp duty, exchange fees), slippage, and minimum lot sizes inflates paper returns by 30–50%.

03

Slow Iteration

Training RL agents for days per experiment destroys research velocity. You can't build intuition about what works without rapid feedback.

04

No Risk Model

Agents that maximize raw returns without drawdown constraints become reckless. Risk must be part of the objective, not an afterthought.

What We Built

Six Core Capabilities
That Address Each Failure Mode

RL-Powered Agents

Soft Actor-Critic (SAC) with twin-Q critics, trained on real NSE data. Continuous action space: simultaneously allocates across 20 stocks.

SAC · JAX · PyTorch 20 stocks / episode

Walk-Forward Validation

5-window rolling validation. The agent must generalize across all windows — not just one period. Val/test gap tracked to detect overfitting.

5-window rolling Consistency >70% gate

GPU-Accelerated Training

XLA-compiled JAX training loop via lax.scan. EXP-002 ran 5M steps in 517 seconds on an RTX 4000 Ada — 9,708 steps/sec.

9,708 SPS · RTX 4000 Ada 517s for 5M steps

NSE-Realistic Simulation

Full Indian brokerage cost model: STT, exchange fees, SEBI charges, GST, stamp duty. Slippage, minimum lot sizes, and market impact included.

Full NSE fee structure Kite Connect API

Multi-Granularity Data

746 stocks × 8 time granularities = 5,567 parquet files. 1-minute to daily. NIFTY 50, 100, 200, 500, Midcap, Smallcap covered.

5,567 parquet files 1min · 3min · 5min · daily

Risk-Aware Reward Design

12-component reward function: drawdown penalties, CVaR, concentration limits, turnover costs, holding incentives. Agent learns risk discipline, not just return maximization.

How It Works

Six-Stage Pipeline
Every Stage Gated

An agent only advances when it clears measurable criteria. No exceptions. This is what separates research-grade from production-grade.

✓
Stage 01Complete

Data Ingestion

746 NSE/BSE stocks across 8 granularities via Kite Connect. Checkpointed parallel downloads, quality validation, multi-source fallback. 5,567 parquet files on disk.

✓
Stage 02Complete

Feature Engineering

Technical indicators (SMA, EMA, RSI, MACD, Bollinger, ATR) + auxiliary signal models: regime detector, volatility forecaster, trend classifier. Observation: 561-dimensional state vector.

✓
Stage 03Complete · EXP-001, EXP-002

RL Agent Training

SAC agents trained with NSE-realistic frictions. Gate: Test Sharpe > 0.4, Win Rate > 55%, Max Drawdown > −12%. EXP-002 cleared all gates: Sharpe 0.4964, Win Rate 65.6%.

●
Stage 04In Progress

Walk-Forward Validation

5-window rolling test proving the agent generalizes — not just memorizes one period. Adversarial stress tests: flash crash (−20%), volatility spike (2× vol), trend reversal.

●
Stage 05In Progress

Paper Trading

30+ day live simulation against real market data. Realistic order fills, portfolio tracking, real-time P&L. An agent that cannot pass paper trading does not go live.

○
Stage 06Planned

Live Deployment

Sub-100ms inference via Zerodha Kite Connect. Live risk controls: hard position limits, drawdown circuit breakers, concentration guards. Real capital only after stages 4 and 5 clear.
